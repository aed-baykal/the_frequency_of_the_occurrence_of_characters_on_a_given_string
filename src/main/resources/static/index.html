Я сделал два способа передачи строки для дальнейшего определения частоты появления символов из которых она состоит.

  1. Можно пересчитать частоту появления символов отправив GET запрос на  http://localhost:8080/rest/{строка}
    где {строка} - это вводимая строка для анализа. 
    У этого запроса много ограничений.
    Существуют зарезервированные символы, которые имеют зарезервированные значения, это разделители — :/?#[]@ — и поддиапазоны — !$&'()*+,;=
    Существует также набор символов, называемых незарегистрированными символами — буквенно-цифровые символы и —._~ - которые не подлежат кодированию.
    Это означает, что все из этих символов, что не относятся к набору незарегистрированных символов, должны быть закодированы в %, когда они передаются как часть параметра GET.

  2. Можно запросом POST отправить JSON на  http://localhost:8080/rest
    вида:
    {
    "symbols" : "{строка}"
    }
    где {строка} - это вводимая строка для анализа.
    Здесь нужно кодировать следующие символы:
    \b Backspace (ascii code 08)
    \f Form feed (ascii code 0C)
    \n New line
    \r Carriage return
    \t Tab
    \" Double quote
    \\ Backslash character
